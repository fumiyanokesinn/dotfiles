# ベースマキナ操作ガイド

事務所（ビジネスパートナー）向け管理画面を運用するためのガイド。

## 環境

| 項目 | 値 |
|------|-----|
| URL | `https://palmu.basemachina.com` |
| 認証 | `./scripts/get-admin-credentials.sh basemachina` |
| ドキュメント | https://docs.basemachina.com |
| ビュー管理リポジトリ | `light-inc/basemachina`（GitHub） |

## 概念モデル

```
プロジェクト
  └── 環境（= 事務所ごと。変数で事務所データを分離）
       ├── ビュー（JSX/TSXで記述されたUI）
       ├── アクション（SQLクエリやAPI呼び出し）
       ├── データソース（MySQL, BigQuery, Google Sheets等）
       └── ナビゲーション（サイドバーメニュー）
```

- 1環境 = 1事務所。現在180以上の環境が存在
- 各環境の変数: `BUSINESS_PARTNER_ID`, `事務所スプレッドシートID` 等

## ロール体系

| ロール | 主な権限 |
|--------|---------|
| プロジェクト管理者 | 全権限 |
| プロジェクトユーザー管理者 | ユーザー・グループ・環境設定 |
| 開発責任者 | アクション・ビュー管理、実行権限設定 |
| 開発者 | アクション・ビュー管理（権限設定不可） |
| アクション運用責任者 | 実行権限・バージョン設定 |

事務所ユーザーは基本的にロールなし（閲覧・実行のみ）。グループで環境アクセスを制御。

## 新規事務所追加フロー

### Step 1: メールアドレス回収
事務所から管理画面ログイン用メールアドレスを取得。

### Step 2: グループ作成
設定 → グループ管理 → 「（事務所名）担当者」グループを作成。

### Step 3: 環境作成
設定 → 環境管理 → 新規環境を作成。
- 環境名: 「（事務所名）様」
- 変数設定: `BUSINESS_PARTNER_ID`, `事務所スプレッドシートID` 等

### Step 4: データソース設定
環境ごとにデータソースを設定（全環境共通ではない）。

### Step 5: ユーザー招待
Palmuプロジェクト → ユーザー管理 → メールアドレスで招待。

### Step 6: グループ割り当て
招待したユーザーを事務所グループに割り当て。

### Step 7: 事務所への案内
ログインURL・マニュアルを送付。

## アクション管理

- SQLアクション: `{{ パラメータ名 }}` で動的値、`{{ vars.BUSINESS_PARTNER_ID }}` で環境変数参照
- ページネーション: `offset`, `limit` パラメータ（システム値）を設定。1ページ最大2000件
- JavaScriptアクション: `executeAction()` で他のアクションを呼び出し
- バージョン管理: 保存ごとに自動バージョン作成。環境別にバージョン固定可能
- ソース管理: `light-inc/basemachina` リポジトリの `actions/` 配下

## ビュー管理

- React + JSX/TSX で記述
- `@basemachina/view` パッケージの hooks を使用
- ソース管理: `light-inc/basemachina` リポジトリの `src/views/` 配下
- Chakra UI 利用可能

## トラブルシューティング

| 症状 | 原因 | 対応 |
|------|------|------|
| データが表示されない | `BUSINESS_PARTNER_ID` の未設定・誤り | 環境の変数設定を確認 |
| 特定ライバーが表示されない | `business_partner_users` に紐付けなし | Admin画面で事務所所属を確認 |
| ログインできない | ユーザー未招待 or グループ未割り当て | Palmuプロジェクトのユーザー管理を確認 |
| エラー表示 | SQLエラー or データソース接続エラー | アクションのクエリ・データソース接続を確認 |
